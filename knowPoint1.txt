在分支功能这一块，需要注意的是，你从从远程克隆的项目一般只能是一个项目的一个分支，如果你需要在同一个项目下的不同项目进行开发操作，那么你就需要从本地同步远程的分支，才行，具体的命令为： git checkout -b dev origin/dev 这样你就能够同步该项目的不同分支的信息了

再来如果你在本地建立了一个远程服务器没有的分支，需要将本地的分支信息同步到远程服务器上，你就需要在建立一个对应的上流分支与本地的这个分支进行相应的关联
对应的指令是 git push --set-upstream origin branchname
其中 --set-upstream 是告诉Git 这是本地的一个新的分支，需要在远程上建立一个新的分支， 后面的origin branch指的是远程在origin对应的库里需要建立的分支的名称，这个名称自然是是要和你本地的新分支的名称保持一致的
